# Governance Model

## Overview

BiblioGenius is an open-source project driven by its community. This document outlines how the project is governed and how decisions are made.

## Roles

### Users

Community members who use the software and provide feedback, bug reports, and feature suggestions.

### Contributors

Members who contribute code, documentation, or other work to the project. Anyone can be a contributor.

### Maintainers

Contributors who have been granted write access to the repository. They are responsible for:

* Reviewing and merging Pull Requests.
* Triaging issues.
* Steering the project direction.

## Decision Making

Decisions are made by consensus among the Maintainers. Major changes are discussed in GitHub Issues or Discussions before implementation.

## Code of Conduct

All participants are expected to adhere to the [Code of Conduct](CODE_OF_CONDUCT.md).
